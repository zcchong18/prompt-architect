# 📜 Refinement Standards: `prompt-architect`

This document defines the standards for all documentation updates related to the `prompt-architect` project. The goal is to ensure consistency, clarity, and high usability.

---

## 1. 🗣️ Tone & Voice
* **Professional yet Enthusiastic:** Act as an expert companion. Use active, impactful verbs (e.g., "Deploy," "Version," "Validate") instead of passive descriptions.
* **Developer-Centric:** Assume familiarity with Python, Git, and CLI environments. Do not over-explain basic terminal commands unless they are project-specific.
* **Punchy & Direct:** Minimize "fluff." Every sentence must either provide information, provide instruction, or establish context.

## 2. 🏗️ Structural Requirements
* **The "F-Pattern" Layout:** Crucial information (Value Prop, Quick Start) must be at the top of the document.
* **Logical Hierarchy:** Follow a standard flow:
    1. `Title & Badges`
    2. `Summary (The Pitch)`
    3. `Problem & Solution`
    4. `Installation`
    5. **`Usage / Quick Start (Mandatory)`**
    6. `Core Features & Tech Stack`
    7. `Roadmap`
* **Heading Consistency:** Use a single level of heading for major sections (e.g., `##`) and sub-sections (e.g., `###`).

## 3. 🧪 Content & Examples
* **The "Copy-Paste" Test:** All code blocks in the `Usage` section must be valid, runnable, and complete. A user should be able to copy a block and see a result.
* **Pre-requisite Clarity:** If a command relies on a specific file structure or environment variable, explicitly document it *immediately before* the code block.
* **Visual Signposting:** Use emojis as anchors (e.g., 🚀 for Installation, 🛠️ for Tech Stack, 💡 for Tips, 🧪 for Testing).

## 4. 📝 Formatting Standards
* **Shields & Badges:** Maintain the use of `shields.io` for Python version, License, and Project Status.
* **Markdown Integrity:** Ensure all links are functional, code fences are properly closed, and lists are correctly indented.
* **Emphasis:** Use **bolding** for key terms and `backticks` for file names, commands, and variables.
